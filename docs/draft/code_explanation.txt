  2. requirements.txt 점검

  현재 상태

  - requirements.txt 파일이 존재하지 않습니다.
  - 백엔드 의존성은 pyproject.toml + Poetry로 관리됩니다.
  - 프론트엔드 의존성은 package.json + pnpm으로 관리됩니다.

  필요성

  ┌──────────────────────────┬─────────────────────────────────────────────────────────┐
  │           상황           │                          권장                           │
  ├──────────────────────────┼─────────────────────────────────────────────────────────┤
  │ Poetry를 사용하는 환경   │ pyproject.toml만으로 충분 (poetry install)              │
  ├──────────────────────────┼─────────────────────────────────────────────────────────┤
  │ Poetry 없이 pip으로 실행 │ requirements.txt 필요                                   │
  ├──────────────────────────┼─────────────────────────────────────────────────────────┤
  │ Docker 빌드              │ Dockerfile이 poetry install을 사용하므로 현재 문제 없음 │
  ├──────────────────────────┼─────────────────────────────────────────────────────────┤
  │ CI/CD                    │ ci.yml에서 poetry install을 사용하므로 현재 문제 없음   │
  └──────────────────────────┴─────────────────────────────────────────────────────────┘

  결론: requirements.txt는 Poetry를 설치하지 않고 pip install -r requirements.txt로 빠르게 환경을
   구성하려는 사용자를 위해 제공하는 것이 좋습니다. poetry export -f requirements.txt --output
  requirements.txt로 생성할 수 있습니다.

  3. 새 환경 생성 절차

  현재 프로젝트에서 새로운 개발 환경을 구성하는 방법은 다음과 같습니다:

  방법 A: Docker Compose (권장, 원클릭)

  git clone <repo> sns-solution && cd sns-solution
  cp .env.example .env         # 환경 변수 편집
  docker compose up -d          # 8개 서비스 전체 기동


  이 방법은 PostgreSQL, Redis, MinIO, Backend, Celery, Frontend, Nginx, MinIO-init가 모두
  자동으로 구성됩니다. 별도 설치가 필요 없습니다.

  방법 B: 로컬 개발 (인프라만 Docker)

  # 1. 인프라 (DB, Redis, MinIO)
  docker compose up -d postgres redis minio minio-init

  # 2. 백엔드
  cd backend
  poetry install                    # 또는: pip install -r requirements.txt
  alembic upgrade head              # DB 마이그레이션
  uvicorn app.main:app --reload     # http://localhost:8000

  # 3. 프론트엔드
  cd ../frontend
  pnpm install
  pnpm dev                          # http://localhost:3000

  # 4. Celery (별도 터미널)
  cd backend
  celery -A app.tasks.celery_app worker --loglevel=info -Q critical,high,medium,low
  celery -A app.tasks.celery_app beat --loglevel=info
